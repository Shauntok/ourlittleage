# 小时代 Project Map

## 项目定位

小时代不是普通博客，而是一个“居民可以长期居住的故事社区”。

核心感觉：
- 温柔
- 深夜
- 有房间感
- 有居民身份
- 有日记、文章、广场、个人主页
- 未来会有等级、徽章、通知、评论、关系系统

正式域名：
- https://www.ourlittleage.com

---

## 当前正式路线

游客：
- `/` Landing Page

居民：
- `/home` 居民首页
- `/diary` 我的日记
- `/diary/new` 写日记
- `/diary/[id]` 查看日记
- `/diary/edit/[id]` 编辑日记
- `/space` 深夜广场
- `/space/diaries` 日记广场
- `/space/articles` 文章广场
- `/posts/[slug]` 文章阅读页
- `/write` 写文章
- `/u/[username]` 公开居民房间
- `/settings/profile` 房间资料设置

后台：
- `/admin`
- `/admin/users`
- `/admin/users/[id]`
- `/admin/reports`
- `/admin/logs`
- `/admin/settings`

## 日记系统

文件：
- `app/diary/page.tsx`
- `app/diary/new/page.tsx`
- `app/diary/[id]/page.tsx`
- `app/diary/edit/[id]/page.tsx`

功能：
- 写日记
- 日记默认 private
- 用户可选择 public
- 最低 11 字
- 每日发布限制
- 编辑日记不自动保存
- 只有手动保存才更新 edited_at / edit_count

规则：
- 草稿系统暂时不用于日记
- 日记日期以发布时刻为准
- 过了 12 点发布就属于新一天

---

## 文章系统

文件：
- `app/write/page.tsx` 或旧文章发布页
- `app/posts/[slug]/page.tsx`
- `app/space/articles/page.tsx`

功能：
- 写文章
- Markdown
- 图片上传
- slug
- tags
- visibility
- draft / published

注意：
- 文章和日记不要混成一个流
- 文章适合长文、故事、作品
- 日记适合碎片、情绪、今天

---

## 广场系统

文件：
- `app/space/page.tsx`
- `app/space/diaries/page.tsx`
- `app/space/articles/page.tsx`

定位：
- `/space` 是总广场入口
- `/space/diaries` 是日记广场
- `/space/articles` 是文章广场

规则：
- 日记流：轻、慢、碎片感
- 文章流：像书架、阅读感更强
- 不要把日记和文章完全混在一起

---

## Admin 系统

文件：
- `app/admin/users/page.tsx`
- `app/admin/users/[id]/page.tsx`

功能：
- 居民管理
- 修改 role
- 修改 status
- 写 admin_logs
- 被警告 / 禁言 / 封禁时发送 notifications

角色：
- owner
- admin
- moderator
- user

规则：
- moderator 不能改身份 / 状态
- admin 不能管理 owner
- admin 不能创建 owner
- owner 不应被别人降级
- owner 不应误降级自己

## 当前重要技术债

1. `/admin/profile` 和 `/settings/profile` 已经重复  
   方向：保留 `/settings/profile`，废弃 `/admin/profile`

2. `avatar` 与 `avatar_url` 曾经混用  
   方向：全部统一为 `avatar_url`

3. `/articles` 和 `/space/articles` 曾经混用  
   方向：
   - 公共文章广场：`/space/articles`
   - 未来我的文章：建议 `/me/articles`

4. `/posts/[slug]` 是文章阅读页  
   日记跳转应该用 `/diary/[id]`

5. 首页所有“首页 / Logo”应该去 `/home`  
   `/` 只保留 Landing Page

## 开发原则

以后每加功能之前，先判断它属于：

- 前台居民系统
- 后台管理系统
- 公共广场系统
- 用户设置系统
- 世界氛围系统

不要重复造系统。  
能融合就融合，能统一字段就统一字段。

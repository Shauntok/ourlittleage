# 小时代 Project Map

## 项目定位

小时代不是普通博客，而是一个“居民可以长期居住的故事社区”。

核心感觉：
- 温柔
- 深夜
- 有房间感
- 有居民身份
- 有日记、文章、广场、个人主页
- 未来会有等级、徽章、通知、评论、关系系统

正式域名：
- https://www.ourlittleage.com

---

## 日记系统

文件：
- `app/diary/page.tsx`
- `app/diary/new/page.tsx`
- `app/diary/[id]/page.tsx`
- `app/diary/edit/[id]/page.tsx`

功能：
- 写日记
- Markdown
- 图片上传
- 日记默认 private
- 用户可切换 public
- 编辑日记
- 日记预览

规则：
- 草稿系统暂时不用于日记
- 日记日期以发布时刻为准
- 过了 12 点发布属于新一天
- 编辑日记不自动保存
- 只有手动保存才更新：
  - `edited_at`
  - `edit_count`

定位：
- 日记适合：
  - 情绪
  - 今天
  - 深夜碎片
  - 没来得及说出口的话

- `app/articles/[id]/page.tsx`
- `app/articles/edit/[id]/page.tsx`

- 编辑文章
- 文章预览

- `/articles` = 我的文章
- `/articles/[id]` = 阅读文章
- `/articles/edit/[id]` = 编辑文章
- `/space/articles` = 公共文章广场

- `published`

visibility：
- `public`

- `hidden`
- `unlisted`

- 不再继续扩展旧 slug route
- 文章和日记共用 `posts` table
- `type` 区分：
  - `article`
  - `diary`

定位：
- 文章适合：
  - 长文
  - 故事
  - 世界观
  - 认真表达
  - 作品

## 广场系统

文件：
- `app/space/page.tsx`
- `app/space/diaries/page.tsx`
- `app/space/articles/page.tsx`

定位：
- `/space` 是总广场入口
- `/space/diaries` 是日记广场
- `/space/articles` 是文章广场

规则：
- 日记流：
  - 轻
  - 慢
  - 碎片感

- 文章流：
  - 更像书架
  - 阅读感更强

注意：
- 不要把日记和文章完全混在一起

## Admin 系统

文件：
- `app/admin/users/page.tsx`
- `app/admin/users/[id]/page.tsx`
- `app/admin/broadcast/page.tsx`

功能：
- 居民管理
- 修改 role
- 修改 status
- 写 admin_logs
- 群发系统信件
- 被警告 / 禁言 / 封禁时发送 notifications

角色：
- owner
- admin
- moderator
- user

规则：
- moderator 不能改身份 / 状态
- admin 不能管理 owner
- admin 不能创建 owner
- owner 不应被别人降级
- owner 不应误降级自己

Broadcast：
- 使用 `/admin/broadcast`
- `announcement` 旧系统未来废弃
- broadcast 仅发送给当前 active 用户
- 新注册用户不应收到历史广播

## 开发原则

以后每加功能之前，先判断它属于：

- 前台居民系统
- 后台管理系统
- 公共广场系统
- 用户设置系统
- 世界氛围系统

不要重复造系统。

能融合就融合。
能统一字段就统一字段。

新增功能优先复用：
- `posts`
- `profiles`
- `notifications`

不要轻易新增重复 table。

小时代现在已经进入：
- 平台架构阶段
而不再只是单页面开发阶段。
